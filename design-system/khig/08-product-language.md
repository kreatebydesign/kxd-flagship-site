# KHIG Part 8 — Product Language

How KXD speaks across the operating system.

**Voice:** Executive. Confident. Calm.  
**Not:** AI chatbot. Enterprise SaaS. Marketing copy.

---

## Voice Principles

| Principle | Why |
|-----------|-----|
| Evidence before assertion | Trust comes from proof |
| Short sentences | Executives scan; they do not read manuals |
| No exclamation marks | Calm confidence, not enthusiasm |
| No "AI" language | "Intelligence", not "AI-powered insights" |
| No hedge words | "May", "might", "consider" — state what is true |
| Client names when relevant | Relationships are personal |

---

## Intelligence

**Role:** Chief of staff briefing.

| Do | Don't |
|----|-------|
| "Business health is healthy. Two priorities need attention." | "Here's your AI-generated dashboard!" |
| "No significant operational issues were detected today." | "All systems go! 🎉" |

**Tense:** Present. Active where possible.

---

## Recommendations

**Role:** Trusted advisor — one clear action.

| Element | Voice |
|---------|-------|
| Title | Verb-first or object-first — "Triage 3 new website reviews" |
| Why appeared | Factual — "3 website reviews remain untriaged." |
| Expected impact | Outcome — "Improves client response time." |
| CTA | "Take action" — not "Let's go!" or "Fix now" |

**Never:** Generic tasks without reason ("Review your inbox").

---

## Insights

**Role:** Observations — not tasks.

| Do | Don't |
|----|-------|
| "Project completion pace increased today with 2 work items finished." | "Great job on completions!" |
| "Operational health has remained stable." | "Everything is awesome!" |

Insights never include CTAs.

---

## Success

| Context | Example |
|---------|---------|
| Work completed | "Work completed · Primal Motorsports" |
| Healthy state | "Operations are clear." |
| No risks | "No actionable risks detected." |

Celebrate through **facts**, not cheerleading.

---

## Warnings

| Level | Voice |
|-------|-------|
| Attention | "3 client threads are awaiting a studio reply." |
| Elevated | "Review volume has increased — response lag may affect trust." |
| Critical | "2 work items are blocked — execution is stalling." |

No ALL CAPS. No red text without a decision behind it.

---

## Errors

| Do | Don't |
|----|-------|
| "Could not save notes." | "Oops! Something went wrong." |
| "Review does not belong to this client." | "Error 400: validation failed" |

State what happened and what the operator can do.

---

## Empty States

| Context | Example |
|---------|---------|
| No changes | "No meaningful changes in the last 48 hours." |
| No priorities | "No urgent priorities — operations are clear." |
| No recommendations | "No action required right now." |
| No risks | "No actionable risks detected." |
| No opportunities | "No opportunities surfaced from current data." |

Empty is honest. Never fake placeholder content.

---

## Confirmations

| Action | Example |
|--------|---------|
| Save | "Saved" (quiet, inline) |
| Copy | "Link copied" |
| Destructive | "Delete this revision? This cannot be undone." |

Confirmations are brief. Destructive actions require explicit language.

---

## Words We Use

| Use | Avoid |
|-----|-------|
| Work | Tasks (client-facing legacy excepted) |
| Revision | Ticket |
| Portfolio | Database |
| Briefing | Dashboard |
| Observation | Insight (UI label "Insight" is OK; voice stays observational) |
| Studio | Team (internal) |
| Client | User (for portal accounts) |

---

## Words We Never Use

- "AI-powered", "smart", "intelligent assistant"
- "Synergy", "leverage", "optimize" (as filler)
- "Oops", "Uh oh", "Whoops"
- "Simply", "Just", "Easily"
- Exclamation marks in operational UI
